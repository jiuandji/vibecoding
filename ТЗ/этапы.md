# 🗓 Этапы реализации

Разбивка работ по `ТЗ/тз.md` на последовательные этапы. Каждый этап даёт работающий промежуточный результат — можно остановиться на любом и получить ценность.

## Этап 1: Фундамент (1-2 сессии)

**Цель:** статичный каркас без 3D.

- [ ] Создать `practice-4/` с `index.html`, `style.css`, `script.js`
- [ ] Настроить CSS-переменные из ТЗ (палитра, типографика)
- [ ] Подключить Google Fonts (Space Grotesk, JetBrains Mono)
- [ ] Сверстать все секции HTML (без 3D, без анимаций)
- [ ] Базовая CSS-стилизация (grid, layout, цвета)
- [ ] Mobile-адаптация через media queries

**Результат:** страница выглядит как ТЗ, но без 3D и интерактива.

---

## Этап 2: Three.js основы (1-2 сессии)

**Цель:** ввести 3D в hero-секцию.

- [ ] Подключить Three.js через CDN или npm
- [ ] Класс `StarField` — базовое поле точечных звёзд (Points + PointsMaterial)
- [ ] Scene + Camera (PerspectiveCamera) + WebGLRenderer
- [ ] Responsive canvas (ресайз окна)
- [ ] Параллакс от мыши (движение камеры на основе mousemove)
- [ ] requestAnimationFrame-цикл

**Результат:** hero с живым 3D-starfield.

---

## Этап 3: Интерактив UI (1 сессия)

**Цель:** подтянуть GSAP-анимации и UX из practice-3.

- [ ] Класс `CustomCursor` — прицел вместо золотого круга
- [ ] Magnetic-поведение для CTA-кнопок
- [ ] IntersectionObserver для scroll-reveal секций
- [ ] Page loader (показать пока Three.js инициализируется)
- [ ] Mobile menu (overlay)

**Результат:** страница ощущается иммерсивно даже без scroll-driven камеры.

---

## Этап 4: Расширенный 3D (2 сессии)

**Цель:** 3D-модели миссий и сложная сцена.

- [ ] Класс `MissionCard` — 3D-превью (сфера/тор с материалом)
- [ ] Освещение (AmbientLight + PointLight)
- [ ] Rotation-on-hover для карточек
- [ ] Класс `TimelineCamera` — движение камеры по Z на скролл
- [ ] Оптимизация: detect mobile, снизить количество частиц

**Результат:** полноценная 3D-секция миссий + scroll-driven timeline.

---

## Этап 5: Форма и счётчики (1 сессия)

**Цель:** остальной интерактив.

- [ ] Класс `CounterAnimator` — анимация чисел при входе в viewport
- [ ] Класс `FormHandler` — валидация, submit, success-state
- [ ] Success-анимация (частицы взлетают, можно через Three.js)

**Результат:** все интерактивные элементы работают.

---

## Этап 6: Полишинг (1 сессия)

**Цель:** production-ready.
