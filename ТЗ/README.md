# 📋 Техническое задание

Формализация идеи `practice-4` из `идея/practice-4.md` до уровня техзадания, готового к реализации.

## Содержание

- [`тз.md`](./тз.md) — основное техническое задание (требования, стек, структура)
- [`этапы.md`](./этапы.md) — план реализации по итерациям (6 этапов)
- [`критерии-приемки.md`](./критерии-приемки.md) — чеклист "готово/не готово"

## Как использовать

1. **`тз.md`** — читать целиком перед стартом. Содержит функциональные и технические требования.
2. **`этапы.md`** — использовать как roadmap. Каждый этап — это одна-две рабочие сессии.
3. **`критерии-приемки.md`** — сверяться в конце каждого этапа и на финальной приёмке.

## Связь с другими папками

```
иследование/  ──►  идея/       ──►  ТЗ/        ──►  practice-4/
(что было)        (что хотим)       (как делать)     (реализация)
```

Каждая следующая папка конкретизирует предыдущую:

| Папка | Уровень абстракции |
|-------|-------------------|
| `иследование/` | Выводы из прошлого |
| `идея/` | Концепция будущего |
| `ТЗ/` | Конкретные требования |
| `practice-4/` | Код (ещё не создано) |
