# 📋 Техническое задание: practice-4 "Cosmos Archive"

**Версия:** 1.0
**Базируется на:** `идея/practice-4.md`, выводах `иследование/`
**Тип:** учебный проект, одностраничный лендинг

---

## 1. Общее описание

**Рабочее название:** Cosmos Archive
**Концепт:** Иммерсивный лендинг вымышленного архива космических миссий с использованием 3D-графики.
**Цель обучения:** освоить Three.js, продолжая линию прогрессии по анимационным технологиям.

### Целевая аудитория (фиктивная, для контекста дизайна)
- Энтузиасты космоса
- Научно-популярная публика
- Студенты астрономических факультетов

---

## 2. Функциональные требования

### 2.1 Обязательные секции

| # | Секция | Описание |
|---|--------|----------|
| 1 | Hero | 3D starfield с параллаксом, название, подзаголовок, CTA |
| 2 | Mission Archive | 3-6 карточек миссий с 3D-превью зондов |
| 3 | Timeline | Горизонтальная шкала 1957-2150, scroll-driven камера |
| 4 | Data Wall | Анимированные счётчики статистики |
| 5 | Launch Control | Форма "подачи заявки на миссию" |
| 6 | Footer | Координаты, частоты, ссылки |

### 2.2 Интерактивные элементы

- **Параллакс starfield** от движения мыши (amplitude: 10-20px)
- **Hover на карточках миссий** — модель зонда вращается
- **Scroll-driven timeline** — камера движется вдоль оси Z при скролле
- **Magnetic CTA-кнопки** (наследуется из practice-3)
- **Custom cursor** в виде прицела с разными состояниями
- **Счётчики чисел** — анимация от 0 до целевого значения при появлении
- **Форма** — валидация, success-state (как в прошлых проектах)

### 2.3 Валидация формы

- **Full Name:** обязательное, минимум 2 символа
- **Email:** обязательное, валидный формат
- **Mission Preference:** select из списка миссий
- **Bio:** textarea, опциональное, максимум 500 символов

## 3. Технические требования

### 3.1 Стек

| Слой | Технология |
|------|-----------|
| Разметка | HTML5, семантические теги |
| Стили | Vanilla CSS, CSS Custom Properties |
| Скрипты | Vanilla JS (ES2022+), классы |
| 3D | Three.js (последняя стабильная версия) |
| UI-анимации | GSAP 3.12+ |
| Шрифты | Space Grotesk, JetBrains Mono (Google Fonts) |

**Запрещено:**
- Фреймворки (React/Vue/Svelte) — не цель этого проекта
- Препроцессоры CSS — поддерживается линия vanilla из прошлых практик
- jQuery и другие legacy-библиотеки

### 3.2 Производительность

- **FPS:** 60 на десктопе, ≥30 на мобильных
- **First Contentful Paint:** < 2 сек на 4G
- **Размер JS-бандла:** ≤ 800KB (Three.js + GSAP + код)
- **Lighthouse Performance:** ≥ 80

### 3.3 Адаптивность

| Брейкпоинт | Ширина | Поведение |
|------------|--------|-----------|
| Mobile | ≤ 640px | Упрощённая 3D-сцена, меньше частиц |
| Tablet | 641-1024px | Полная функциональность, адаптированный layout |
| Desktop | ≥ 1025px | Полная функциональность |

### 3.4 Браузерная поддержка

- Chrome/Edge: последние 2 версии
- Firefox: последние 2 версии
- Safari: последние 2 версии
- WebGL: обязательно, с graceful fallback на статичный фон

### 3.5 Архитектура JS

```
script.js
├── class StarField           // 3D-поле звёзд
├── class MissionCard         // 3D-превью зондов
├── class TimelineCamera      // управление камерой на скролле
├── class CustomCursor        // кастомный курсор-прицел
├── class CounterAnimator     // анимация счётчиков
├── class FormHandler         // обработка формы
└── App                       // координация всего
```

Минимум 6 классов — продолжение линии practice-3 (5 классов).

## 6. Функциональные сценарии (user journeys)

### Сценарий 1: первый визит
1. Пользователь открывает страницу
2. Видит загрузчик (1-2 сек, инициализация Three.js)
3. Hero появляется с 3D-звёздами и magnetic CTA
4. Скроллит дальше, камера начинает движение

### Сценарий 2: просмотр миссии
1. Наводит курсор на карточку миссии
2. 3D-модель зонда начинает вращаться
3. Кликает — скроллится к подробному описанию (опционально в v2)

### Сценарий 3: подача заявки
1. Скроллит до Launch Control
2. Заполняет форму
3. Отправляет — видит success-анимацию (частицы взлетают)
