# 🎨 Требования к дизайну

Детальная дизайн-спецификация для practice-4 "Cosmos Archive". Синтез из трёх предыдущих папок:

```
иследование/дизайн-системы.md  ─┐
идея/practice-4.md              ├──►  дизайн/  (вы здесь)
ТЗ/тз.md                        ─┘
```

## Содержание

- [`визуальный-язык.md`](./визуальный-язык.md) — настроение, палитра, типографика, сетка, форма, тени
- [`компоненты.md`](./компоненты.md) — спецификация UI-элементов (nav, кнопки, карточки, формы, курсор, loader, footer)
- [`движение.md`](./движение.md) — принципы анимации, длительности, easing, ключевые последовательности

## Как использовать

1. **Перед началом вёрстки** — прочитать `визуальный-язык.md`, скопировать CSS-переменные в проект.
2. **При создании компонентов** — сверяться с `компоненты.md`, не изобретать заново.
3. **При добавлении анимаций** — использовать токены длительности и easing из `движение.md`.
4. **При сомнениях** — держать в голове "настроение" из раздела 1 `визуальный-языка.md`: научный романтизм, не sci-fi экшн.

## Ключевые вещи, которые легко забыть

- **Свечение, а не тень** — `box-shadow` всегда окрашен в акцентный цвет, не чёрный
- **Моноширинный для цифр** — JetBrains Mono, не Space Grotesk для числовых данных
- **Почти без скруглений** — `--radius-sm` (4px) это потолок для 90% элементов
- **5% акцентов** — если видно больше двух nebula-элементов в viewport, что-то лишнее
- **`prefers-reduced-motion`** — проверяется обязательно, не опционально
