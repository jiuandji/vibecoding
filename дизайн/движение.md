# 🎬 Движение и анимации

Motion design для practice-4. Определяет темп, ритм и характер движения на всей странице.

## 1. Принципы движения

### 1.1 Сдержанность
Движение должно подчёркивать содержание, а не соперничать с ним. Никаких "прыгающих" элементов, избыточного bounce-а.

### 1.2 Осмысленность
Каждая анимация должна иметь назначение:
- Направить взгляд (scroll reveal)
- Подтвердить действие (button press)
- Показать состояние (loading, success)
- Создать атмосферу (starfield parallax)

### 1.3 Инерция
Движение в космосе — медленное и плавное. Никаких резких остановок.

---

## 2. Длительности (duration tokens)

```css
--duration-instant: 100ms;   /* hover подчеркивания */
--duration-fast:    200ms;   /* small UI transitions */
--duration-base:    400ms;   /* стандартные переходы */
--duration-slow:    800ms;   /* scroll-reveal, большие элементы */
--duration-epic:    1600ms;  /* hero-анимации, loader */
```

**Правило:** чем крупнее элемент, тем длиннее анимация.

---

## 3. Easing (кривые)

```css
--ease-out-expo:    cubic-bezier(0.19, 1, 0.22, 1);     /* выход "выстрелом" */
--ease-in-out-cubic:cubic-bezier(0.65, 0, 0.35, 1);     /* симметричный */
--ease-out-quart:   cubic-bezier(0.25, 1, 0.5, 1);      /* плавный out */
--ease-elastic:     GSAP elastic.out(1, 0.5)            /* магнитные кнопки */
```

### Когда что использовать

| Ситуация | Кривая |
|----------|--------|
| Появление элемента | `--ease-out-expo` |
| Переход между состояниями | `--ease-in-out-cubic` |
| Hover-эффекты | `--ease-out-quart` |
| Magnetic buttons / bounce | `--ease-elastic` (GSAP) |
| Scroll-driven | `linear` (синхронизация со скроллом) |

---

## 4. Базовые переходы на элементах

### 4.1 Универсальный transition
```css
* {
  transition: color var(--duration-fast) var(--ease-out-quart),
              background-color var(--duration-fast) var(--ease-out-quart),
              border-color var(--duration-fast) var(--ease-out-quart);
}
```

Покрывает 80% случаев. Дополнительные transitions — на конкретных компонентах.

### 4.2 Transform и opacity
- Используются для **всех** entry-анимаций (не `top/left`, не `display`)
- Причина: GPU-ускорение, 60 FPS

## 5. Ключевые анимации

### 5.1 Page Load Sequence (первые 2 секунды)

```
0ms     → Loader появляется
300ms   → Progress bar начинает расти
1200ms  → Three.js готов, loader начинает fade-out
1600ms  → Loader полностью скрыт, hero-анимация стартует
1700ms  → Starfield появляется (fade + scale)
2000ms  → Hero-заголовок (text scramble + fade)
2400ms  → CTA-кнопки появляются (stagger 100ms)
```

### 5.2 Scroll Reveal

Каждая секция при входе в viewport:
```
opacity: 0 → 1
translateY: 40px → 0
duration: --duration-slow (800ms)
easing:   --ease-out-expo
```

**Threshold IntersectionObserver:** 0.15 (10-15% секции в viewport → триггер).

### 5.3 Staggered появление внутри секции

Элементы одного типа (карточки, list items) появляются с задержкой 80-120ms друг от друга.

```js
cards.forEach((card, i) => {
  card.style.transitionDelay = `${i * 100}ms`;
});
```

### 5.4 Timeline Camera Scroll

- Прогресс скролла секции Timeline (от 0 до 1) → позиция камеры по Z
- Плавная интерполяция: `camera.position.z = lerp(camera.position.z, target, 0.1)` в RAF-цикле
- Точки на timeline подсвечиваются когда `Math.abs(camera.position.z - point.z) < threshold`

### 5.5 Starfield Parallax

- `mousemove` → target position
- В RAF: `current.x += (target.x - current.x) * 0.05` (smooth follow)
- Применяется к `camera.position.x` и `camera.position.y`
- Amplitude: 15-20px максимум

### 5.6 Mission Card Hover

```
Hover in:
  border-color:     --color-horizon → --color-nebula   (200ms)
  box-shadow:       none → --glow-soft                 (400ms)
  3D-модель:        starts rotating (1 оборот / 8 сек)

Hover out:
  reverse в обратном порядке
  3D-модель дорисовывает оборот до целого числа
```

### 5.7 Counter Animation

При входе в viewport:
```
0 → target value
duration: 2000ms
easing:   ease-out-quart
```

Используется `requestAnimationFrame`, не `setInterval`.

### 5.8 Form Success

При успешной отправке:
1. Fields fade out (200ms)
2. Success-экран fade in (400ms)
3. Bonus: частицы "взлетают" с места формы вверх за пределы экрана через Three.js (эффект запуска)

## 6. Micro-interactions

### 6.1 Magnetic Buttons (из practice-3)

```
mousemove над кнопкой:
  dx = (e.clientX - buttonCenterX) * 0.3
  dy = (e.clientY - buttonCenterY) * 0.3
  gsap.to(button, { x: dx, y: dy, duration: 0.4, ease: "power2.out" })

mouseleave:
  gsap.to(button, { x: 0, y: 0, ease: "elastic.out(1, 0.5)", duration: 0.8 })
```

### 6.2 Text Scramble (для hero-заголовка)

- На загрузке: буквы появляются через "расшифровку" из случайных символов
- Длительность: 40ms per step, общая длительность ~1200ms
- Набор символов: `!<>-_\\/[]{}—=+*^?#`
- Переосмысление: добавить "космические" глифы (◊ ▲ ◆ ○) в набор

### 6.3 Input Focus Glow

```
Focus in:
  border-bottom переход от --color-horizon к --color-nebula (200ms)
  под инпутом: дополнительная линия выезжает слева направо (scale-x 0→1, 300ms, transform-origin left)
```

## 7. Что НЕ делаем

- Нет `animation-iteration-count: infinite` для декоративных элементов (кроме starfield и loader — всё, что пульсирует, делает это с причиной)
- Нет `bounce`-эффектов на крупных элементах (только на magnetic buttons — маленькое, осмысленное движение)
- Нет параллакса фона от скролла — только от мыши (избегаем дешёвого "веб-2.0" эффекта)
- Нет анимаций длительностью > 2 секунд, кроме page load sequence
- Нет GIF / видео-фонов — всё движение процедурное

## 8. Производительность

### 8.1 Обязательные проверки
- DevTools > Performance: 60 FPS на всех анимациях
- `will-change: transform` на элементах, которые анимируются regularly
- `transform: translateZ(0)` для форсирования GPU при необходимости

### 8.2 Условное отключение

```js
const prefersReducedMotion = window.matchMedia('(prefers-reduced-motion: reduce)').matches;
if (prefersReducedMotion) {
  // отключить parallax, scramble, magnetic buttons
  // оставить только fade-in / fade-out
}
```

### 8.3 Mobile-оптимизация
- Starfield: уменьшить количество частиц в 3 раза
- Отключить custom cursor
- Отключить magnetic buttons (touch-устройства их не чувствуют)
- Timeline camera scroll: оставить, но упростить (меньше точек)
