# 🌌 Practice-4: Cosmos Archive

**Концепт:** Иммерсивный лендинг-архив для вымышленного космического агентства или музея миссий. Тема — глубокий космос, путешествие сквозь звёздное поле.

## Почему именно это

Вывод из `иследование/техники-анимации.md`: прогрессия шла
```
Canvas функциональный → Canvas ООП → GSAP + классы → ???
```

Логичный следующий шаг — **Three.js / WebGL**. Это:
- Продолжение анимационной линии, а не пивот
- Новый ментальный инструмент (3D-координаты, камера, свет)
- Один новый ментальный шаг — как и было во всех прошлых проектах

Вывод из `иследование/дизайн-системы.md`: дизайн-оси уже пройдены
```
Light (сакура) → Warm dark (огонь) → OLED luxury (часы) → ???
```

Предлагаемое направление — **Deep Space** (глубокий синий/фиолетовый + белое свечение звёзд). Сохраняем контрастную темную эстетику `practice-3`, но меняем характер: не минимализм, а **глубина**.

## Дизайн

| Параметр | Значение |
|----------|----------|
| Тема | Глубокий космос, звёздная глубина |
| Палитра | `#050716` (void), `#6B5CFF` (nebula purple), `#E8EAF0` (starlight) |
| Шрифты | Space Grotesk (заголовки), JetBrains Mono (тех. данные) |
| Стиль | Слои, параллакс, эффект глубины |
| Библиотеки | Three.js + GSAP (продолжаем использовать) |

## Секции

1. **Starfield Hero** — 3D поле звёзд с параллаксом от мыши, название агентства
2. **Mission Archive** — 3-6 карточек миссий (Voyager-4, Europa Clipper, etc.) с 3D-моделями зондов
3. **Timeline** — горизонтальная временная шкала (1957 → 2150), скролл-управление камерой
4. **Data Wall** — статистика (км пройдено, планет посещено) с анимацией чисел
5. **Launch Control** — форма "записи на миссию" (пародия на заявку)
6. **Footer** — координаты, частоты радиосвязи (стилизованный текст)

## Новые ментальные инструменты

- **Three.js основы:** Scene, Camera, Renderer, Mesh
- **3D-освещение:** AmbientLight, PointLight для звёзд
- **Shaders (базовые):** fragment shader для свечения звёзд
- **Scroll-driven camera:** движение камеры в 3D-пространстве от скролла
- **Post-processing:** эффект bloom для свечения

## Что сохраняем из прошлых проектов

- GSAP для UI-анимаций (magnetic buttons из practice-3)
- IntersectionObserver для scroll-reveal
- Custom cursor (можно переосмыслить как "прицел")
- CSS-переменные как дизайн-токены
- Пара шрифтов (heading + body)

## Риски

- **Three.js крутая кривая** — может быть слишком большим скачком. Митигация: начать с простого starfield, не пытаться делать полный 3D-сцен сразу.
- **Производительность на мобильных** — 3D тяжёлый. Митигация: детектирование устройства, упрощённая версия без shaders для мобильных.
- **Размер бандла** — Three.js ~600KB. Митигация: использовать tree-shaking, импортировать только нужные модули.
