# 🔀 Альтернативные направления

Если Three.js-путь (`practice-4.md`) кажется слишком крутым скачком или не резонирует — вот другие обоснованные варианты следующего шага.

## Альтернатива A: SPA / Dashboard

**Концепт:** Аналитический интерфейс — не лендинг, а продуктовый UI. Например, "FinTrack" — трекер финансов, или "Habitus" — трекер привычек.

**Почему:** Смена **типа работы** (лендинг → приложение). В прошлых проектах учились продающему UI, пора освоить продуктовый.

**Новые инструменты:**
- Первый фреймворк (React / Vue / Svelte)
- Компонентное мышление
- State management (useState → Context / stores)
- Клиентская маршрутизация
- Chart.js / Recharts для графиков

**Тема:** продуктивный минимализм, data-heavy UI

**Подходит если:** хочется освоить фреймворк, двигаться в сторону реальной разработки приложений.

---

## Альтернатива B: Shader Art / Generative

**Концепт:** Арт-лендинг, где весь фон — GLSL shader. Например, "Fluid Studio" — агентство с фоном из жидкости, реагирующей на курсор.

**Почему:** Вывод из `иследование/анализ-прогресси.md` — визуальная ось усложнялась (light → dark → OLED). Shader art — это уход в **художественное программирование**, не просто анимации.

**Новые инструменты:**
- GLSL (vertex/fragment shaders)
- Noise functions (Perlin, Simplex)
- WebGL через Three.js или shadertoy-подход
- Возможно: PixiJS вместо Three.js

**Тема:** жидкости, туман, органические формы

**Подходит если:** хочется прокачать чисто визуальную сторону, а не архитектуру.

---

## Альтернатива C: Portfolio Meta-Site

**Концепт:** Собрать все 3 готовых проекта в единый портфолио-сайт. Главная страница с превью, роутинг между проектами, общая дизайн-система.

**Почему:** Это уже есть в списке "Следующих шагов" в корневом `README.md`. Прагматичный выбор — **закрепить результат**, а не бросаться в новое.

**Новые инструменты:**
- Статический генератор (Astro / 11ty) ИЛИ SPA роутер
- Vercel / Netlify деплой
- Система превью-карточек
- SEO-оптимизация

**Тема:** мета-дизайн, объединяющий три разных эстетики

**Подходит если:** хочется получить готовый артефакт для демонстрации, а не ещё один учебный проект.

## Сравнение

| Критерий | Practice-4 (3D) | SPA | Shader | Portfolio |

| Сложность | высокая | высокая | очень высокая | средняя |

| Продолжает линию анимаций | да | нет | да | нет |

## Рекомендация

Если цель — **обучение по траектории анимаций** (как в прошлых проектах), делать основной `practice-4.md` (3D).
Если цель — **применимость в реальной работе**, идти в SPA или Portfolio.
