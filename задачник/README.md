# 📓 Задачник / Отчёт

Финальная папка подготовительного этапа. Содержит:
- **Отчёт** — что было сделано в папках `иследование/`, `идея/`, `ТЗ/`, `дизайн/`
- **Задачник** — конкретные задачи для реализации practice-4 с чекбоксами

## Содержание

- [`отчет.md`](./отчет.md) — сводка проделанной работы по четырём папкам
- [`задачи.md`](./задачи.md) — 90 атомарных задач для реализации practice-4

## Позиция в цепочке

```
иследование/  ──►  идея/  ──►  ТЗ/  ──►  дизайн/  ──►  задачник/  ──►  practice-4/
(рефлексия)        (концепт)   (что)       (как)        (план действий)    (код)
```

Задачник — это **мост между планированием и реализацией**. Дальше — только код.

## Как использовать

### Если нужно быстро понять, что сделано
Читай `отчет.md` — за 5 минут получишь полную картину подготовительного этапа.

### Если готов начать кодить
Открывай `задачи.md` и иди сверху вниз по чекбоксам. Каждая задача — 15-60 минут.

### Если застрял на задаче
Сверься с соответствующей папкой:
- Вопрос про архитектуру → `ТЗ/тз.md`
- Вопрос про визуал → `дизайн/компоненты.md` или `дизайн/визуальный-язык.md`
- Вопрос про анимацию → `дизайн/движение.md`
- Вопрос про последовательность → `ТЗ/этапы.md`

## Что дальше

1. **Открыть `задачи.md`**
2. **Начать с Этапа 0** (создание папки practice-4)
3. **Отмечать чекбоксы** по мере выполнения
4. **Коммитить после каждого этапа** (если будет git)
5. **Сверяться с `ТЗ/критерии-приемки.md`** в конце каждого этапа
