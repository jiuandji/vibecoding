## 1. Папка `иследование/`

**Что сделано:** анализ трёх существующих проектов (practice-1, practice-2, practice-3) с трёх углов.

### Файлы и ключевые выводы

**`анализ-прогресси.md`**
- Прогрессия построена не по объёму кода, а по качеству техник
- Три оси усложнения: визуальная (light → OLED), анимационная (Canvas → GSAP), UX (статика → иммерсив)

**`техники-анимации.md`**

- IntersectionObserver — сквозной инструмент во всех проектах

**`дизайн-системы.md`**
- Типографическая пара (Playfair / Cormorant + Inter) работает универсально
- Border-radius как индикатор "теплоты" проекта (24px → 0px)

## 2. Папка `идея/`

**Что сделано:** предложена концепция следующего проекта (practice-4) с альтернативами.

### Основная идея: Cosmos Archive
- **Тема:** архив космических миссий, "научный романтизм"
- **Стек:** Three.js + GSAP (продолжение анимационной линии)
- **Дизайн:** deep space (новая визуальная ось после OLED)
- **Обоснование:** логичное продолжение обеих прогрессий из research

### Альтернативы (на случай если основная не подойдёт)
- SPA / Dashboard — смена типа работы (лендинг → приложение)
- Shader Art — уход в художественное программирование
- Portfolio Meta-Site — закрепить результат и собрать готовые проекты

## 4. Папка `дизайн/`

**Что сделано:** детальная дизайн-спецификация.

### Файлы и содержание

**`визуальный-язык.md`**
- Настроение: научный романтизм (не sci-fi экшн)
- Палитра: 7 цветов (void / abyss / nebula / plasma / starlight / dust / horizon) + свечения
- Типографика: Space Grotesk + JetBrains Mono, fluid sizing через clamp()
- Сетка: 4px spacing scale
- Форма: почти без скруглений (max 8px)
- Запреты: нет градиентов, нет material-тени, нет ярких цветов

**`компоненты.md`** — 8 компонентов:
- Nav (desktop + mobile)
- Кнопки (primary / ghost / icon)
- Карточки (mission / data)
- Формы (input / label / textarea / select / submit)
- Custom Cursor (как прицел)
- Timeline
- Page Loader
- Footer

**`движение.md`**
- 3 принципа: сдержанность, осмысленность, инерция
- Duration tokens: 100ms (instant) → 1600ms (epic)
- Easing: 4 кривые + GSAP elastic
- Ключевые последовательности: page load, scroll reveal, timeline camera, starfield parallax
- Правило `prefers-reduced-motion` — обязательно

## Логическая целостность

Что важно — каждая папка **опирается на предыдущую**, а не дублирует её:

- `иследование/` → выводы из фактов (что уже сделано)
- `идея/` → концепция основана на выводах
- `ТЗ/` → требования основаны на концепции
- `дизайн/` → визуал основан на требованиях

Это означает: при реализации practice-4 все решения имеют обоснование, прослеживаемое на 3 шага назад.
