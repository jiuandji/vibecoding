# 🎬 Техники анимации в проектах

Исследование анимационных подходов, использованных в трёх лендингах.

## Canvas 2D — процедурная анимация

Используется в `practice-1` и `practice-2` для фоновых эффектов частиц.

### practice-1: лепестки сакуры
Функциональный подход — массив объектов `{x, y, rotation, speed}`, обновление в цикле `requestAnimationFrame`. Физика: гравитация + горизонтальный дрейф + вращение.

### practice-2: огоньки (Embers)
ООП подход — класс `Ember` с методами `update()` и `draw()`. Частицы поднимаются вверх, а не падают вниз. Это шаг от процедурного мышления к объектному.

**Наблюдение:** оба случая решают ту же задачу (анимация N частиц), но учат разным парадигмам.

## IntersectionObserver — scroll-анимации

Присутствует во всех трёх проектах, но с разной сложностью:

| Проект | Техника |
|--------|---------|
| practice-1 | Базовое `observe()` + добавление класса `visible` |
| practice-2 | Staggered delay — карточки появляются с задержкой друг за другом |
| practice-3 | Комбинация с CSS `--ease-out-expo` переменными |

## GSAP — библиотечная анимация (только practice-3)

### Magnetic Buttons
Кнопка "притягивается" к курсору:
- Слушатель `mousemove` на кнопке
- Расчёт смещения от центра
- `gsap.to()` с `ease: "elastic.out"` для возврата

### Text Scramble
Буквы "дешифруются" из случайных символов:
- Массив символов-мусора
- Интервал, заменяющий случайные буквы
- Прогрессивное "раскрытие" реального текста

### Custom Cursor
Кастомный курсор, следующий за мышью:
- `requestAnimationFrame` + lerp (линейная интерполяция)
- Разные состояния: default / hover / click
- `mix-blend-mode: difference` для контраста на любом фоне

| Курсор | нативный | нативный | кастомный |

## Паттерны, которые повторяются

1. **RequestAnimationFrame loop** — основа всех Canvas и кастомных анимаций
2. **IntersectionObserver** — универсальный инструмент scroll-эффектов
3. **CSS transitions** для простых ховеров (везде)
4. **Класс-based архитектура** — появляется в practice-2, углубляется в practice-3
